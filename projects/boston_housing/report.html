Implementation: Define a Performance Metric¶

It is difficult to measure the quality of a given model without quantifying its performance over training and testing. This is typically done using some type of performance metric, whether it is through calculating some type of error, the goodness of fit, or some other useful measurement. For this project, you will be calculating the coefficient of determination, R2, to quantify your model's performance. The coefficient of determination for a model is a useful statistic in regression analysis, as it often describes how "good" that model is at making predictions.

The values for R2 range from 0 to 1, which captures the percentage of squared correlation between the predicted and actual values of the target variable. A model with an R2 of 0 is no better than a model that always predicts the mean of the target variable, whereas a model with an R2 of 1 perfectly predicts the target variable. Any value between 0 and 1 indicates what percentage of the target variable, using this model, can be explained by the features. A model can be given a negative R2 as well, which indicates that the model is arbitrarily worse than one that always predicts the mean of the target variable.

For the performance_metric function in the code cell below, you will need to implement the following:

  • Use r2_score from sklearn.metrics to perform a performance calculation between y_true and y_predict.
  • Assign the performance score to the score variable.

from sklearn.metrics import r2_score

def performance_metric(y_true, y_predict):
    """ Calculates and returns the performance score between 
        true and predicted values based on the metric chosen. """
    
    # Calculate the performance score between 'y_true' and 'y_predict'
    score = r2_score(y_true, y_predict)
    
    # Return the score
    return score

Question 3 - Training and Testing¶

What is the benefit to splitting a dataset into some ratio of training and testing subsets for a learning algorithm?
Hint: What could go wrong with not having a way to test your model?

Answer: Our goal is to create a model which generalizes well so that we can predict outcomes for future data points that are not part of our training data, so naturally we would like a way of testing how well our model will generalize once we start using it to make predictions using new data. Setting aside a fraction of the data to be our testing set accomplishes this goal. We train our models using the training data and then we test how well the models generalize by making predictions with the testing set and checking our error metrics. The major risk of not doing a train/test split is that your model could overfit the training data and then generalize very poorly to new instances outside of the training set. In other words, if error metrics for the model are only collected against the training data that the model learns on, it may achieve great accuracy, even 100%, but then perform very poorly once it is applied to new data.

Answer: With a max depth of 1, the model suffers from high bias. With a max depth of 10, the model suffers from high variance, or overfitting. This is because we have given our decision tree too much freedom to map the complexities of the training data. The main visual cues for diagnosing the bias and variance have to do with the difference in results between the training and testing scores. This tells us that we just aren't modeling the data that well, but the part that we are modeling is generalizing. In the max_depth=10 high variance case we have a very high (if not perfect) score on the training data, but the testing score is much lower. This distance tells us that while we are modeling the training set very closely, we are overfitting and not generalizing well outside of the training data.

Question 7 - Grid Search¶

What is the grid search technique and how it can be applied to optimize a learning algorithm?

Answer: The grid search technique allows us to test different combinations of hyperparameter values for our models so that we can choose parameters which give us the best fit to our data. For example, if we were training an SVM we might like to play with the kernel, C, and gamma values. Rather than inefficiently tuning these manually and adjusting little by little based on outcomes, the grid search technique creates a grid which is the cross product of all of the possible combinations of the parameters we give it. It then creates a model for each possible combination, trains and tests it against our data, and determines which model performs the best.

Question 8 - Cross-Validation¶

What is the k-fold cross-validation training technique? What benefit does this technique provide for grid search when optimizing a model?
Hint: Much like the reasoning behind having a testing set, what could go wrong with using grid search without a cross-validated set?

Answer: The k-fold cross-validation technique is where we partition the data into k "folds". We then perform a k step learning process in which k-1 folds are used for training the model and the remaining fold is used for validation (a different one each time). From here, we can compute the average error from our k steps.

Applying this technique to grid search, we need a method for checking our error in order to reliably choose the best combination of values for our hyperparameters, and k-fold cross-validation is one CV method we can use for this. If we performed grid search without using some sort of CV, we could end up choosing hyperparameter values that work well on training data but generalize poorly to unseen data. Since k-fold allows each data point to be used for validation once and for training k-1 times, the variance in the error is much lower and we can have more confidence in the hyperparamters chosen by grid search if it is using k-fold CV (or another CV method!).

from sklearn.metrics import make_scorer
from sklearn.tree import DecisionTreeRegressor
from sklearn.grid_search import GridSearchCV

def fit_model(X, y):
    """ Performs grid search over the 'max_depth' parameter for a 
        decision tree regressor trained on the input data [X, y]. """
    
    # Create cross-validation sets from the training data
    cv_sets = ShuffleSplit(X.shape[0], n_iter = 10, test_size = 0.20, random_state = 0)

    # Create a decision tree regressor object
    regressor = DecisionTreeRegressor()

    # Create a dictionary for the parameter 'max_depth' with a range from 1 to 10
    params = {'max_depth': np.arange(1,11)}

    # Transform 'performance_metric' into a scoring function using 'make_scorer' 
    scoring_fnc = make_scorer(performance_metric)

    # Create the grid search object
    grid = GridSearchCV(regressor, params, scoring=scoring_fnc, cv=cv_sets)

    # Fit the grid search object to the data to compute the optimal model
    grid = grid.fit(X, y)

    # Return the optimal model after fitting the data
    return grid.best_estimator_
